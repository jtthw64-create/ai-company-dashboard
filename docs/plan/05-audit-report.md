## 1. 감사 방법과 커버리지 맵

이 감사는 **원격 Claude Code 세션**(격리 컨테이너)에서 수행되었다. 접근 가능 여부 자체가 시스템의 관측성을 보여주는 데이터이므로 그대로 기록한다.

| 영역 | 상태 | 확인 방법 |
|---|---|---|
| repo 3개 대시보드 + README + git 전체 히스토리(24커밋) | ✅ 검증됨 | 직접 정독, file:line 인용 |
| Google Drive `Claude Cowork GD`(전역 CLAUDE.md, settings.local.json, launch.json) | ✅ 검증됨 | Drive 읽기 도구(승인 불필요) |
| claude.ai 스킬 12종·플러그인 5종·커넥터 10종 **목록/설명문** | ✅ 검증됨 | ListSkills/ListPlugins/ListConnectors |
| Notion 워크스페이스 | ✅ 표본 검증 | 검색 다수 실행 |
| **Gist 3종 데이터 실물** (company-status / teamrooms / church-orch) | ⛔ UNVERIFIED | 원격 프록시가 gist 원문 차단(403 실측) — iMac 또는 브라우저에서 검증 필요 |
| **Dropbox 전체** (AI Company/, Claude Dropbox/, worklog, gist-config.md) | ⛔ UNVERIFIED | 이 세션에서 도구 호출마다 승인 게이트 — Cowork 세션에 위임 |
| **스킬 12종 본문(SKILL.md)** | ⛔ UNVERIFIED | claude.ai는 설명문만 노출, 원본은 iMac 로컬(위치 불확실 — 사용자 확인) |
| **iMac 로컬** (~/.claude, 로컬 폴러 구현체, launchd/cron, Obsidian LLM WIKI 볼트) | ⛔ UNVERIFIED | 원격에서 접근 불가 — Cowork 세션에 위임 |

> UNVERIFIED 영역의 검증 절차는 전부 `06-fix-commands.md`의 **V-시리즈(위임 검증 명령)** 로 작성되어 있다.
