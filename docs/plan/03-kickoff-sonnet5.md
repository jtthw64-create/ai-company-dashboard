## 작업 규율
- 커밋: 단계당 1커밋 이상, 메시지는 단계 ID 접두("P2-S3: ..."). force-push 금지. push는 git push -u origin <브랜치>, 네트워크 실패 시 2/4/8/16초 재시도.
- push 전 시크릿 스캔(필수): grep -rInE '(ghp|github_pat)_[A-Za-z0-9]{8,}' docs/ skills/ roles/ ops/ wiki-templates/ devices/ 2>/dev/null — 걸리면 push 중단·제거 후 재시도. Gist ID 하드코딩은 대시보드 HTML에만 허용(기존 상태), 신규 문서에는 금지.
- 기록: 단계 완료마다 07의 표에 [단계 ID | 상태 | 커밋 SHA | 증거(명령 출력 붙여넣기)] 추가. 07 이외의 계획 문서(00~06, 08)는 수정하지 않는다(00~06 수정이 필요하면 07에 제안 기록).
