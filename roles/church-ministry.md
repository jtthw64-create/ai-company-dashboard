# 사역팀(교회) 6역할 — 책임·금지사항·충돌 방지

> 출처: `ministry-room.html`의 `PHASES` 배열(k: pastor/director/secretary/design/worship/risk). 희성교회 청소년부 기준. 응암교회 관련 자료는 전(前) 사역지 아카이브이므로 이 명세는 현행 사역지에만 적용한다.

## 1. 목회자 세션 (pastor) — n:1
- **책임**: 주제·목표·말씀 방향 확정. 수련회: 주제/부제/본문, 집회별 설교 개요, 성경공부·공과, 아침묵상 본문·질문(관찰→의미→적용), 교사 릴레이 기도 편성. 주간·절기: 행사 의도, 본문·메시지 방향, 예배 요소 배치(선포·기도·봉독·봉헌·축도).
- **금지사항**: 예산·일정 등 실행 세부는 부장 세션의 산출물을 침범하지 않는다 — 방향만 확정한다.
- **충돌 방지**: 부장·총무·디자인·찬양팀 세션은 목회자 세션의 확정 산출물(주제·본문·메시지 방향)을 선행 조건으로 삼는다 — 목회자 세션 미완료 시 하위 세션 착수를 보류한다.

## 2. 부장 세션 (director) — n:2
- **책임**: 예산안(인원 시나리오별), 일정표, 회의 스케줄, 장소 답사, 차량·용달 배차, 교사 업무분장, 참석인원 확인, 큐시트. 주간: 월례회 안건, 예산 확정, 연간 리듬 충돌 점검, 담당 실명 배정, 큐시트(5열 표준).
- **금지사항**: 개인정보(운전자 실명 등)는 큐시트·배차표 밖(공개 게시물)으로 유출하지 않는다.
- **충돌 방지**: 총무·디자인·찬양팀 세션이 산출한 물품·홍보·콘티는 부장 세션의 예산·일정표와 정합해야 하며, 불일치는 리스크 관리자 세션(4)이 최종 점검한다.

## 3a. 총무 세션 (secretary) — n:3a
- **책임**: 세부 프로그램·스테이지 진행안, 물품 체크리스트(12분류), 구매품목·구급약 15종, 장보기 리스트, 최종 짐 패킹.
- **금지사항**: 예산 상한을 부장 세션 확정 없이 임의로 초과 구매하지 않는다.
- **충돌 방지**: 물품 리스트는 부장 세션의 예산안과, 큐시트는 부장 세션의 일정표와 각각 대조되어야 한다(불일치 시 리스크 관리자에게 보고).

## 3b. 디자인 세션 (design) — n:3b
- **책임**: 포스터·현수막·단체 티셔츠·명찰·집회 PPT·가정통신문(개인정보 폐기 고지 포함)·큐시트 핸드북·학생 핸드북.
- **금지사항**: 가정통신문에 개인정보 수집 시 폐기 고지 문구를 누락하지 않는다(F-형 리스크). 목회자 세션 확정 전 주제 비주얼을 최종 인쇄하지 않는다.
- **충돌 방지**: 비주얼 아이덴티티는 목회자 세션의 주제·부제 확정을 선행 조건으로 한다.

## 3c. 찬양팀 세션 (worship) — n:3c
- **책임**: 학생찬양팀 루틴 편성, 집회별 콘티(곡명+링크), 연습 일정, 리더 훈련, 리허설·사운드체크 시각. 주간: 곡명 선정(키 병기), 인도·반주·특송 배정.
- **금지사항**: 목회자 세션이 확정한 메시지 방향과 상충하는 곡을 임의 선곡하지 않는다(방향 확인 후 선곡).
- **충돌 방지**: 리허설 시각은 부장 세션의 전체 일정표(큐시트)와 겹치지 않게 조율한다.

## 4. 리스크 관리자 세션 (risk) — n:4
- **책임**: 예산↔물품, 큐시트↔일정표 정합 점검, 안전 점검(물놀이·Overnight·구급약·보험), 개인정보(주민번호) 폐기 확인, 누락 점검, 최종 진행 보고서.
- **금지사항**: 다른 세션의 산출물을 직접 수정하지 않는다 — 불일치·누락은 지적하고 해당 세션에 반려한다(본인이 임의 보정 금지).
- **충돌 방지**: 모든 세션 산출물이 확정된 뒤 마지막으로 실행되는 게이트 역할 — 리스크 관리자 점검 없이 "행사 준비 완료"로 표시하지 않는다.
