# sermon-loop — 설교 글쓰기 Loop

> `docs/plan/02-master-plan.md` P4-S1. 강의의 "글쓰기 Loop"(초안→리뷰→수정→재검토) 개념을 설교 준비에 적용한 것. `roles/church-ministry.md`의 목회자 세션(pastor) 산출물 흐름과 연결된다.
> Writing Reviewer 서브에이전트(P3-S4, 연구팀과 공유되는 동형 부품)가 아직 별도 스킬로 존재하지 않는 세션에서는, 이 스킬 안의 "리뷰 체크리스트"(3단계)를 직접 자기 검토 기준으로 사용한다 — Writing Reviewer 스킬이 편입되면 3단계를 그 스킬 호출로 교체한다(파일 구조는 바뀌지 않음).

## 언제 쓰는가
- 사용자(목회자)가 본문·주제를 확정한 뒤, 설교 원고를 준비할 때.
- 입력: 확정된 본문(passages), 주제/부제, 대상(주간 예배 vs 수련회 집회).

## 4단계

### 1. 본문 확정 (Text)
- `roles/church-ministry.md` 목회자 세션 산출물(본문·부제·메시지 방향)을 그대로 입력으로 받는다. 이 단계에서 새로 본문을 바꾸지 않는다(확정된 것을 존중).

### 2. 초안 (Draft)
- 본문 관찰→의미→적용 구조로 초안을 작성한다(아침묵상 질문 설계와 동일 3단 구조 — 사역팀 관례 재사용).
- 산출물: `draft-<날짜>.md`.

### 3. 리뷰 (Review — Writing Reviewer 체크리스트)
1. **논리**: 관찰→의미→적용의 각 단계가 실제로 이어지는가(비약 없음)?
2. **근거**: 본문 밖 주장(교리·적용)에 본문적 근거가 명시되어 있는가? 원문 없는 단정은 없는가?
3. **대상 적합성**: 청소년부 대상 언어·예화가 적절한가?
- 산출물: `review-<날짜>.md`(위 3항목에 대한 지적 목록, 통과/보완 표시).

### 4. 수정 → 재검토 (Revise → Re-review)
- 리뷰 지적사항을 반영해 `revised-<날짜>.md` 작성.
- 3단계 체크리스트를 재적용해 통과 확인(`check-<날짜>.md`). 2회 연속 같은 항목에서 실패하면 자동 반복하지 않고 목회자 세션에 보고(전역 CLAUDE.md 2회 실패 규칙과 동형 적용).

## 산출물 저장 경로
- Dropbox 사역 폴더(경로 표준은 FIX-09/devices/ 확정 후 반영 — 그 전까지는 임시로 `Claude Cowork GD/사역/설교/<날짜>/`에 4개 파일: `draft`, `review`, `revised`, `check`).

## 드라이런 예시 (P4-S1 수용 기준 — 샘플 본문 1개, 1사이클)

**입력(예시)**: 본문 — 시편 23편, 대상 — 주간 예배, 메시지 방향 — "주님이 목자 되심을 신뢰하는 삶"(형식 시연용 예시, 실제 설교 원고 아님).

1. **Text**: 시편 23편, 목회자 세션 확정 사항으로 간주.
2. **Draft**(`draft-2026-07-12.md` 예시): 관찰(목자-양 비유의 고대 근동 배경) → 의미(하나님의 돌보심에 대한 신뢰) → 적용(청소년 일상의 불안 속 신뢰 훈련 한 가지 제안).
3. **Review**(`review-2026-07-12.md` 예시): 논리=통과 / 근거=통과(본문 인용 명시) / 대상 적합성=보완 필요("고대 근동 배경" 설명이 청소년 대상엔 다소 길다는 지적).
4. **Revise→Re-review**(`revised-2026-07-12.md`, `check-2026-07-12.md` 예시): 배경 설명 축약, 적용부 구체 사례 1개 추가 → 재검토 3항목 모두 통과.

> 위 예시는 스킬 형식·Loop 동작을 시연하기 위한 것이며 실제 설교 원고 산출물이 아니다.
