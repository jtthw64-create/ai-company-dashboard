# CLAUDE.md — AI Company 상황판 (이 repo)

이 문서는 이 repo(`ai-company-dashboard`)에서 작업하는 모든 세션(로컬 CLI·원격 claude.ai/code·Cowork)의 공통 규약이다. 계획 문서(`docs/plan/`)와 함께 읽는다 — 특히 진행 중인 빌드가 있으면 `docs/plan/00-README.md`부터.

## 1. 시스템 지도

- **대시보드 3장** (GitHub Pages, 이 repo가 소스):
  - `index.html` — 회사 상황판(팀 현황·알림·브리핑·접수함)
  - `team-room.html` — 팀별 회의실(개발·연구·사업)
  - `ministry-room.html` — 사역팀 회의실(교회 행사 오케스트레이션 + 기획 보드)
- **Gist 3종** (비공개, 상태 저장소 — ID·토큰은 이 repo에 절대 기록하지 않는다. `Dropbox: Claude Dropbox/AI Company/gist-config.md` 참조):
  1. 회사 상황판 접수함(`company-status.json`) — `requests[]`(요청 큐)·`teams[]`(팀 상태)·`worklog[]`·`alerts[]`
  2. 팀 회의실 룸 데이터(`ROOMS_GIST_ID`)
  3. 교회 행사 데이터(`church-orch-data.json`) — 사역팀 전용, events·phases·board·research
- **디스패처(로컬 폴러)**: nurisimac(사용자의 iMac)에 상주하는 실행 엔진. 1분 폴링으로 접수함(`requests[]`)의 `status=="new"` 항목을 감지해 처리하고, 정기 회차가 04:00/18:00/23:00 KST에 추가로 돈다. 이 repo에는 코드가 없다 — 실행 지점은 iMac뿐이다. 상세 명세는 `docs/ops/ENGINE.md`(**TBD — FIX-01 완료 후 채움**).

## 2. 표준 용어

- **"디스패처(로컬 폴러)"** — 이 repo·문서 전체의 표준 명칭. 과거 문서·주석에 등장하는 "오케스트레이터", "폴러" 등은 전부 동의어다(디스패처와 다른 실체가 아님).

## 3. 타임스탬프 규약

- **저장**: KST(`+09:00`) 오프셋 ISO 문자열로 기록한다 (예: `2026-07-12T13:00:00+09:00`).
- **비교**: ISO 문자열을 직접 `<`/`>`/정렬 비교하지 않는다 — 반드시 epoch(ms)로 환산 후 비교한다(대시보드 코드의 `tms()` 패턴 참조).
- **과거 데이터 혼재 사실**: 일부 과거 Gist 기록은 UTC(`Z`) 오프셋으로 남아있다. 코드는 이미 epoch 비교라 안전하지만, 문자열 그대로 비교하는 신규 코드를 작성하지 않는다.

## 4. 안전 보류 기준

다음은 자동 실행하지 않는다 — 항상 "사용자 실행" 전환 + 사유 기록:
- git 커밋 서명(gpg) 변경
- 금전(매매·송금·결제·계약)
- 파일/데이터 삭제(휴지통 이동 포함) — 대체 원칙: 정리 후보로 집계만 하고 사용자 개별 승인 대기
- 외부(타인) 발송 — 이메일·문자·메신저·SNS 게시
- 새 외부 서비스 가입

전역 규약(`전역 CLAUDE.md`, Cowork 동기화 폴더)의 "지시 우선순위" 섹션과 일치한다 — 무인/자율 실행이 명시된 작업 범위 안에서는 "불명확하면 질문" 원칙보다 이 문서·해당 작업 지시문이 우선한다.
